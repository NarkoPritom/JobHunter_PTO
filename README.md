# JobHunter AI

Telegram-бот, который ищет вакансии под ваш профиль (Руководитель/Начальник ПТО,
Руководитель сметного отдела), оценивает их через LLM и помогает откликаться.

## Что реально работает

- Поиск через **публичное HH API** (без ключа) — по умолчанию включено.
- Фильтр по зарплате (от 220 000 ₽) и локации (Москва/ЮВАО/Люберцы/Котельники/
  Жулебино/Некрасовка/Октябрьский/гибрид/удаленно).
- Оценка вакансии через Claude (score 0-100%, причины, плюсы, минусы, красные флаги).
- Карточка вакансии в Telegram с кнопками: Подробнее / Сохранить / Откликнуться / Пропустить.
- Генерация персонального сопроводительного письма под конкретную вакансию.
- **Подтверждение перед отправкой** — бот показывает черновик письма и ждёт вашего
  тапа "Отправить", а не отправляет автоматически без вашего ведома. Это осознанное
  решение: одна и та же формулировка не должна улетать пачкой без проверки.
- SQLite-хранилище (vacancies/favorites/responses/settings/logs), APScheduler
  каждые 30 минут (настраивается).
- Docker + Railway конфиги — деплоится сразу.

## Что НЕ включено и почему

- **Avito Работа**: официального API нет, скрапинг сайта нарушает их ToS —
  не стал это делать. Ссылки на вакансии с Avito можно пересылать боту вручную,
  анализатор и генератор писем работают с произвольным текстом вакансии, не
  только с результатами API (см. `app/llm/analyzer.py`).
- **Автоматическая отправка отклика на HH без вашего токена**: чтобы бот реально
  подавал заявку от вашего имени, HH требует OAuth-токен именно вашего аккаунта
  (это их ограничение, не моё) — см. `HH_ACCESS_TOKEN`/`HH_RESUME_ID` ниже.
  Без них бот всё равно ищет, оценивает и пишет письма, просто финальную отправку
  вы делаете вручную по готовому письму.
- **SuperJob**: клиент есть (`app/superjob/client.py`), но полный пайплайн
  скоринга подключен только для HH — по образцу в `app/scheduler/jobs.py`
  расширяется за 10-15 строк.
- Alembic-миграции: пока используется `create_all()` при старте. Если будете
  часто менять схему БД, добавьте `alembic init` — заготовка в requirements.txt есть.

## Установка

```bash
git clone <ваш-репозиторий>
cd jobhunter-ai
cp .env.example .env   # заполните токены
pip install -r requirements.txt
python main.py
```

### Получение токенов

1. **Telegram bot token**: напишите @BotFather в Telegram → `/newbot`.
2. **Telegram chat id**: напишите своему боту `/start`, затем откройте
   `https://api.telegram.org/bot<TOKEN>/getUpdates` и найдите `"chat":{"id": ...}`.
3. **Anthropic API key**: https://console.anthropic.com → API Keys.
4. **HH токен (опционально, для авто-заполнения отклика)**: https://dev.hh.ru →
   зарегистрировать приложение → пройти OAuth → получить access_token. Плюс
   `resume_id` — id вашего резюме на hh.ru (виден в URL резюме).

## Деплой на Railway

```bash
railway login
railway init
railway up
```

Затем в Railway → Variables добавьте все переменные из `.env.example`.

## Профиль кандидата

Заполнен в `app/services/profile.py` на основе вашего резюме. Если резюме
изменится — просто отредактируйте поля в этом файле (имя, навыки, опыт,
минимальная зарплата, локации).

## Структура проекта

```
jobhunter-ai/
  app/
    config/        # настройки из переменных окружения
    database/       # SQLAlchemy модели и сессии
    repositories/    # доступ к данным
    services/        # профиль кандидата, фильтрация
    llm/             # Claude API: скоринг вакансий, письма
    hh/              # HeadHunter API клиент
    superjob/        # SuperJob API клиент (опционально)
    telegram_bot/    # хендлеры, клавиатуры, меню
    scheduler/        # периодический поиск
    utils/            # логирование
  tests/
  main.py
  Dockerfile / Procfile / railway.json
  requirements.txt / .env.example
```
